# 1. 오픈소스 소프트웨어(OSS) 개요

오픈소스 소프트웨어(Open Source Software, OSS)는 소스코드를 누구나 열람·수정·재배포할 수 있도록 공개한 소프트웨어를 의미한다.
폐쇄형 소프트웨어와 달리, 개발자와 이용자가 자유롭게 기능을 개선하고 버그를 고칠 수 있다는 점에서 혁신을 빠르게 만들어내는 특징이 있다.

대표적인 예로 리눅스, 파이어폭스, 안드로이드 운영체제 등이 있으며 오늘날 대부분의 IT 인프라가 OSS 기반으로 운영될 정도로 중요성이 커지고 있다.

# 2. OSS의 발전 역사
● 1970~1980: 자유 소프트웨어 운동(FSF)

리처드 스톨먼(Richard Stallman)의 Free Software Movement가 등장하며 소프트웨어 자유(사용·연구·수정·배포권리)를 강조했다. GNU 프로젝트가 대표적인 사례이다.

● 1998: “오픈소스(Open Source)” 용어 등장

자유 소프트웨어가 기업에 ‘반기업적’ 이미지로 비쳐지는 문제를 해결하기 위해 “Open Source”라는 용어가 만들어졌다.
오픈소스 이니셔티브(OSI)가 설립되며, OSS가 공식적인 개발 모델로 인정받기 시작했다.

● 2000 이후: Git과 GitHub의 등장

리눅스 토발즈가 만든 Git(2005)과 GitHub(2008)의 등장으로 전 세계 개발자들이 협업할 수 있는 구조가 구축되었다.
이후 오픈소스의 개발 속도와 규모는 폭발적으로 증가하였다.

# 3. 대표적인 오픈소스 소프트웨어 소개
● Linux

전 세계 서버 대부분이 사용하는 운영체제. 커뮤니티가 활발하고 다양한 배포판이 존재한다.

● Python

데이터 분석, AI, 웹 개발 등 다양한 분야에서 사용되는 언어. 풍부한 라이브러리 생태계를 기반으로 성장했다.

● Firefox

Mozilla 재단이 만든 웹 브라우저. 보안성·개인정보 보호 성능이 우수하다.

● Android

구글이 공개한 모바일 운영체제. 스마트폰 OS 시장의 대부분을 차지한다.

# 4. OSS 라이선스(저작권)

오픈소스라고 해서 완전 자유롭게 사용하는 것은 아니며, 라이선스 규칙을 따라야 한다.

대표적 OSS 라이선스는 다음과 같다:

GPL: 수정·배포 시 소스코드 공개 의무 (강한 카피레프트)

MIT: 매우 자유로운 라이선스. 출처만 명시하면 거의 모든 사용 허용

Apache 2.0: 특허 보호 조항 포함. 기업에서 많이 사용

BSD: 최소한의 제한만 존재하는 자유로운 라이선스

라이선스를 준수하는 것은 OSS 생태계 유지에 매우 중요하다.
